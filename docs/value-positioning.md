## 불편 및 니즈 분석 요약

### 공급자 (원장/강사) Pain Points & Needs

| Pain Points | Needs |
|-------------|-------|
| AI 생성 문제의 낮은 퀄리티와 오류 | **Zero-Entry**: 채점/기록의 완전 자동화 |
| 데이터 입력을 위한 추가 노동 발생 | **Hyper-Local**: 우리 동네 기출 완벽 반영 |
| 지역/학교별 출제 경술(Local Context) 반영 부족 | **Customization**: AI 초안 + 인간의 섬세한 편집 |
| 강사의 고유한 편집 권한 및 스타일 무시 | **Retention**: 에이스 강사의 노하우 시스템화 |

### 수요자 (학부모/학생) Pain Points & Needs

| Pain Points | Needs |
|-------------|-------|
| 숫자 위주의 영혼 없는 학습 리포트 | **Action Plan**: 점수 분석을 넘어선 실천 가이드 |
| 태블릿 학습의 지루함과 딴짓 유발 | **Emotional Care**: 학습 동기 부여와 격려 |
| "왜?"라는 질문에 대한 기계적 반복 답변 | **Visual Feedback**: 텍스트가 아닌 직관적 설명 |
| 성적으로 이어지지 않는 단순 문제 풀이 양치기 | **Human Connection**: 전문가의 실시간 개입/관리 |

## 기존 솔루션 vs 현장의 괴리 (Gap Analysis)

| 분석 항목 | 기존 솔루션의 접근 | 현장의 실제 요구 | 해결 점수 |
|----------|------------------|-----------------|----------|
| 문제 생성 | 무한한 문제 생성 (Quantity) | '나올 법한' 고퀄리티 문제 (Quality) | △ |
| 데이터 관리 | 관리자를 위한 대시보드 (Admin) | 실무자를 위한 편리한 UX (Worker) | ✕ |
| 개별화 | 정답률 기반 레벨링 (Level) | 오답의 원인과 사고 과정 분석 (Logic) | △ |

## 핵심 미해결 과제: "The Human-Centric Bridge"

AI는 '정답'과 '통계'는 잘 뽑아내지만, 다음의 세 가지 **맥락(Context)**을 연결하지 못하고 있음:

### 1. 워크플로우의 맥락 (Workflow Context)
- 새로운 서비스에 접속해서 데이터를 넣는 것은 또 다른 업무
- **"사용자가 이미 쓰고 있는 도구(엑셀, 노션, 카톡)"** 안에서 AI가 조용히 작동해야 함
- 자꾸 자기네 플랫폼으로 들어오라고 강요하면 안 됨

### 2. 지역/관계의 맥락 (Relationship Context)
- 분당의 내신과 해남의 학습 환경은 다름
- 강사와 학생 사이의 신뢰 관계를 AI가 자꾸 끼어들어 방해함
- AI는 **'선생님을 더 권위 있게'** 만들어줘야 하는데, 자꾸 '선생님을 대체'하려 함

### 3. 해석의 맥락 (Interpretative Context)
- "80점입니다"라는 데이터는 중요하지 않음
- "그래서 오늘 고생한 아이에게 어떤 말을 해줘야 하는지"
- "이 강사를 계속 믿고 맡겨도 되는지"에 대한 **인간적인 해석**을 제공하지 못함

## Value Positioning 문장

### 통합 포지셔닝 (B2B + B2C)

> "우리 제품은 **중소형 학원 원장/강사와 그들의 학생·학부모**가 **내신 시험 준비와 일상 학습 관리**에서 **AI가 모든 걸 대체하려는 기존 에듀테크**보다 **선생님의 기존 워크플로우(엑셀, 카톡, 노션) 안에서 조용히 작동하며, 지역/학교별 맥락을 반영하고, 숫자가 아닌 '다음에 뭘 해야 하는지' 실천 가이드를 제공하는 방식으로** 더 잘 **선생님은 수업 준비 시간을 줄이고, 학부모는 아이와 대화할 거리를 얻으며, 학생은 '왜 틀렸는지' 이해하는** 문제를 해결한다."

### B2B (학원/강사) 관점

> "우리 제품은 **중소형 학원 원장과 강사**가 **내신 시험 대비 문제 출제와 학생 관리** 상황에서 **새로운 플랫폼 학습을 강요하는 기존 문제은행**보다 **이미 쓰는 도구(엑셀, 카톡) 안에서 작동하고, 우리 동네 기출을 완벽 반영하며, AI 초안 위에 내 스타일로 편집할 수 있게** 더 잘 **수업 준비 시간 단축과 강사 노하우의 시스템화**를 해결한다."

### B2C (학부모/학생) 관점

> "우리 제품은 **성적 정체로 불안한 중학생 학부모와 자기주도학습 고등학생**이 **모르는 문제 해결과 취약점 파악** 상황에서 **풀이만 보여주고 끝나는 기존 AI 튜터**보다 **'왜 틀렸는지' 시각적으로 설명하고, 다음에 뭘 공부해야 하는지 실천 가이드를 주며, 필요할 때 전문가가 개입하는 방식으로** 더 잘 **성적 향상으로 이어지는 진짜 학습**을 해결한다."

## 핵심 차별화 키워드

| 기존 솔루션의 함정 | 우리의 접근 |
|------------------|-----------|
| 새 플랫폼 강요 | **Zero-Entry**: 기존 도구 안에서 작동 |
| 무한 문제 양산 | **Hyper-Local**: 우리 동네 기출 반영 |
| AI가 선생님 대체 | **Assistant**: 선생님을 더 돋보이게 보조 |
| 숫자 중심 리포트 | **Action Plan**: 실천 가이드 제공 |
| 텍스트 풀이만 | **Visual Feedback**: 직관적 설명 |
