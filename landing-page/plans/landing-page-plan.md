# EduFlow 랜딩 페이지 계획

## 개요

EduFlow 프로젝트를 소개하는 GitHub Pages 기반 랜딩 페이지입니다.

**목적**:
- 프로젝트 소개 및 가치 전달
- 잠재 사용자(학원 원장/강사, 학부모/학생)에게 핵심 기능 안내
- 초기 관심자 확보

---

## 기술 스택

| 항목 | 선택 | 이유 |
|-----|-----|-----|
| 프레임워크 | Next.js 14 (Static Export) | 메인 프로젝트와 동일 스택, SSG 지원 |
| 스타일링 | Tailwind CSS | 빠른 개발, 반응형 지원 |
| 애니메이션 | Framer Motion | 부드러운 스크롤 애니메이션 |
| 아이콘 | Lucide React | 가벼움, 일관된 디자인 |
| 배포 | GitHub Pages | 무료, 간편한 CI/CD |

---

## 페이지 구조 (Single Page)

### 1. Hero 섹션
- **헤드라인**: "AI가 선생님을 대체하지 않습니다. 더 돋보이게 만듭니다."
- **서브 헤드라인**: "중소형 학원을 위한 AI 기반 문제 생성 & 학습 관리 플랫폼"
- **CTA 버튼**: "시작하기", "더 알아보기"
- **히어로 이미지/일러스트**: 학원 선생님과 AI가 협업하는 모습

### 2. 문제 제기 (Pain Points) 섹션
- **타이틀**: "이런 고민, 익숙하시죠?"
- 3개 카드 형태:
  - 📝 "매주 수십 종 교재 문제 선별에 수 시간..."
  - 📊 "숫자만 나열된 학습 보고서, 학부모님께 뭐라고 설명하죠?"
  - 🏫 "우리 동네 기출은 반영 안 되고 엉뚱한 문제만..."

### 3. 솔루션 (핵심 차별화) 섹션
- **타이틀**: "EduFlow는 다릅니다"
- 5개 핵심 가치 카드:

| 키워드 | 설명 |
|--------|------|
| Zero-Entry | 새 플랫폼 강요 ❌ → 기존 도구(엑셀, 카톡) 안에서 작동 |
| Hyper-Local | 무한 문제 양산 ❌ → 우리 동네 기출 완벽 반영 |
| Assistant | AI가 선생님 대체 ❌ → 선생님을 더 돋보이게 보조 |
| Action Plan | 숫자 중심 리포트 ❌ → "다음에 뭘 해야 하는지" 실천 가이드 |
| Visual Feedback | 텍스트 풀이만 ❌ → 직관적인 시각적 설명 |

### 4. 핵심 기능 (Features) 섹션
- **타이틀**: "선생님의 시간을 되찾아 드립니다"
- 3단계 플로우 시각화:
  1. **AI 문제 생성**: 단원, 난이도, 학교 선택 → Gemini 기반 생성 → 멀티 LLM 검수
  2. **간편한 배포**: 문자로 학생에게 전송 → 손글씨 풀이 사진 업로드 → OCR 자동 분석
  3. **스마트 분석**: 자동 채점 → 취약점 분석 → 학부모용 "대화 거리" 보고서

### 5. 타겟 사용자 섹션
- **타이틀**: "누구를 위한 서비스인가요?"
- 탭 또는 토글 형태:
  - **학원 원장님**: "65명 학생 관리, 밤 11시까지 채점... 이제 그만하세요"
  - **1인 과외 선생님**: "수업 준비 시간 > 실제 수업 시간? 역전시켜 드립니다"
  - **학부모님**: "학원 다녀도 뭘 배우는지 모르셨다면"
  - **학생**: "오답노트 3일 만에 포기? 자동으로 만들어 드립니다"

### 6. 워크플로우 데모 섹션
- **타이틀**: "이렇게 작동합니다"
- 애니메이션 또는 GIF로 핵심 플로우 시연:
  ```
  [강사] 조건 입력 → [AI] 문제 생성 → [강사] 편집 →
  [시스템] 문자 발송 → [학생] 풀이 제출 →
  [AI] 분석 → [시스템] 보고서 전달
  ```

### 7. 기술 스택 섹션 (개발자/투자자 대상)
- **타이틀**: "검증된 기술로 만들었습니다"
- 로고 배열: Next.js, Supabase, Gemini, OpenAI, Google Cloud Vision

### 8. CTA 섹션
- **헤드라인**: "선생님의 노하우를 시스템화하세요"
- **서브**: "지금 시작하면, 다음 학기가 달라집니다"
- **버튼**: "무료로 시작하기", "문의하기"

### 9. Footer
- 프로젝트 링크 (GitHub)
- 연락처
- 저작권 표시

## 디자인 가이드라인

### 컬러 팔레트
| 용도 | 색상 | 코드 |
|-----|------|-----|
| Primary | 교육적인 블루 | #3B82F6 |
| Secondary | 신뢰의 네이비 | #1E3A5F |
| Accent | 활력의 오렌지 | #F97316 |
| Background | 부드러운 화이트 | #F8FAFC |
| Text | 다크 그레이 | #1E293B |

### 타이포그래피
- **헤드라인**: Pretendard Bold, 48px (Desktop) / 32px (Mobile)
- **서브헤드**: Pretendard SemiBold, 24px / 18px
- **본문**: Pretendard Regular, 16px / 14px

### 톤앤매너
- 따뜻하고 신뢰감 있는 어조
- 전문 용어 최소화, 일상 언어 사용
- "~하세요" 형태의 친근한 존칭
