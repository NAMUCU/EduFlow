프로젝트 개요
EduFlow - AI 기반 학원 운영 자동화 플랫폼 MVP
한국어 교육 서비스로, 학원 강사가 AI로 문제를 생성하고 학생에게 배포하며, 학생 풀이를 분석하는 플랫폼입니다.

기술 스택
* Framework: Next.js 14 (App Router)
* DB/Auth/Storage: Supabase
* UI: Tailwind CSS + shadcn/ui
* 언어: 한국어 (모든 UI 텍스트)

디자인 가이드
* 모던하고 깔끔한 느낌
* 화이트 베이스 + 블루 계열 포인트 컬러
* 충분한 여백, 카드 기반 레이아웃
* 폰트: Pretendard 또는 시스템 한글 폰트

핵심 기능 (MVP 범위)
1. 인증 시스템
* Supabase Auth 사용
* 회원가입/로그인 (이메일)
* 역할 구분: 강사(teacher) / 학생(student)
2. 강사용 기능
2-1. 대시보드
* 등록된 학생 수, 최근 과제, 제출 현황 요약
2-2. 학생 관리
* 학생 등록 (이름, 학교, 학년)
* 학생 목록 조회/수정/삭제
2-3. 문제 생성 (AI 연동은 Mock)
* 입력 폼: 과목, 단원, 난이도, 문제 수
* MVP에서는 AI 호출 없이 Mock 데이터로 문제 생성
* 생성된 문제 편집 기능
* 문제 저장 (Supabase)
2-4. 과제 배포
* 저장된 문제지 선택
* 학생 선택 (다중 선택)
* 배포 버튼 → assignments 테이블에 저장
* 문자 발송은 MVP에서 제외 (UI만 구현, 실제 발송 X)
2-5. 제출 확인 및 분석
* 학생별 제출 현황 조회
* 제출된 풀이 이미지 보기
* OCR/분석은 MVP에서 제외 (업로드된 이미지만 표시)
3. 학생용 기능
3-1. 대시보드
* 받은 과제 목록, 제출 현황
3-2. 과제 풀이
* 배포된 과제 목록 조회
* 문제 보기
* 풀이 이미지 업로드 (Supabase Storage)
* 제출 완료 처리

데이터베이스 스키마 (Supabase)
-- 사용자 프로필
profiles (
  id uuid PK (auth.users 연결),
  email text,
  name text,
  role text ('teacher' | 'student'),
  created_at timestamp
)

-- 학생 정보 (강사가 등록)
students (
  id uuid PK,
  teacher_id uuid FK → profiles,
  name text,
  school text,
  grade text,
  created_at timestamp
)

-- 문제
problems (
  id uuid PK,
  teacher_id uuid FK → profiles,
  subject text,
  unit text,
  difficulty text,
  question text,
  answer text,
  created_at timestamp
)

-- 문제지
problem_sets (
  id uuid PK,
  teacher_id uuid FK → profiles,
  title text,
  problem_ids uuid[],
  created_at timestamp
)

-- 과제 배포
assignments (
  id uuid PK,
  problem_set_id uuid FK → problem_sets,
  student_id uuid FK → students,
  status text ('pending' | 'submitted'),
  created_at timestamp
)

-- 제출물
submissions (
  id uuid PK,
  assignment_id uuid FK → assignments,
  image_url text,
  submitted_at timestamp
)

페이지 구조
/                     → 랜딩 페이지
/login                → 로그인
/signup               → 회원가입

/teacher              → 강사 대시보드
/teacher/students     → 학생 관리
/teacher/problems     → 문제 생성/관리
/teacher/assignments  → 과제 배포
/teacher/submissions  → 제출 확인

/student              → 학생 대시보드
/student/assignments  → 과제 목록/풀이 제출

MVP 제외 항목 (추후 Claude Code에서 구현)
다음 기능은 UI만 구현하거나 Mock 처리:
1. AI 문제 생성 → Mock 데이터 반환 (실제 Gemini 연동 X)
2. 문자 발송 (Tally) → 버튼만 구현 (실제 발송 X)
3. OCR (Google Cloud Vision) → 이미지 업로드만 (분석 X)
4. PDF 변환 (PDF.co) → 제외
5. 멀티 LLM 검수 → 제외
6. RAG 검색 → 제외
7. 학습 보고서 자동화 → 제외

구현 우선순위
1. Supabase 연결 + Auth 설정
2. 프로필/역할 기반 라우팅
3. 강사: 학생 CRUD
4. 강사: 문제 생성 (Mock) + 저장
5. 강사: 과제 배포
6. 학생: 과제 조회 + 이미지 업로드
7. 강사: 제출 확인

참고사항
* 모든 텍스트는 한국어로 작성
* 에러 메시지, 버튼, 라벨 등 모두 한국어
* 반응형 디자인 (모바일 대응)
* 추후 외부 API 연동을 위해 서비스 레이어 분리 권장
